## Code Quality Quick Reference

### Python

- Follow **PEP 8** style guide
- Use **type hints** for all functions
- Add **docstrings** to all classes and functions
- Target **80%+ test coverage**
- Use **Black** for formatting
- Use **isort** for import sorting
- Use **flake8** for linting

### TypeScript/React

- Follow **Airbnb JavaScript Style Guide**
- Use **TypeScript** for type safety
- Add **JSDoc comments** to interfaces and complex functions
- Target **70%+ test coverage**
- Use **Prettier** for formatting
- Use **ESLint** for linting

### General Principles

1. **Write Clean Code** - Self-documenting, readable, maintainable
2. **Test Your Code** - Unit tests, integration tests where appropriate
3. **Document Your Code** - Comments for "why", not just "what"
4. **Follow Conventions** - Consistent naming, structure, patterns
5. **Handle Errors** - Graceful degradation, proper logging
6. **Review Your Code** - Self-review before submitting PR
